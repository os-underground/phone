/ñp ------ ### Sin JS: pausar animación en hover (útil en galerías) Añade esto para que cuando el usuario ponga el mouse sobre el elemento la animación se pause y muestre el color actual. ```css .element:hover { animation-play-state: paused; } ``` --- ### Version combinada: aplicar mismo hue/velocidad a varios elementos (sin duplicar keyframes) - Usa una variable `--hue` actualizada por JS y construye colores con HSL y calc() en CSS. Esto te permite sincronizar bordes y textos cambiando solo `--hue` en el root. ```css :root{ --hue:0; --dur:1s; } .border-sync{ border:3px solid hsl(var(--hue) 90% 55%); transition: border-color .12s linear; } .text-sync{ color: hsl(var(--hue) 90% 55%); transition: color .12s linear; } ``` JS (simple loop hue): ```js let hue = 0; const root = document.documentElement; const speed = 1; // grados por frame o ajusta según prefieras function tick(){ hue = (hue + speed) % 360; root.style.setProperty('--hue', hue); requestAnimationFrame(tick); } tick(); ``` Este enfoque permite que el cambio sea continuo y que controles velocidad centralmente. --- ### Consejos prácticos y accesibilidad - Si el color animado es decorativo, asegúrate de que el contenido principal (texto) tenga contraste suficiente en todo momento. - Evita cambios extremadamente rápidos que puedan causar molestia o desencadenar fotosensibilidad; velocidades razonables: 3–8s por ciclo para fondos y bordes; 1–3s si buscas efecto energético. - Ofrece control (pausar/ajustar velocidad) cuando la animación es prominente en la UI. - Para producción, usa HSL para manipular matiz (hue) fácilmente y para mantener saturación/luminosidad constantes. --- Si quieres, te creo un archivo HTML completo que incluya: - Controls para velocidad y pausa, - Ejemplos de border estático con cambio de color, - Texto con color animado y stroke animado, - Integración con la galería previa (si quieres que las cards y los títulos compartan la misma paleta dinámica). ¿Te lo genero ya listo para pegar en tu repo?
